## Feature 026: ACAS/Nessus Scan Import

ATO Copilot adds **ACAS/Nessus .nessus file import** with automatic **NIST 800-53 control mapping**, **POA&M auto-generation**, and **import history tracking**. Teams can export vulnerability scan results from ACAS (Assured Compliance Assessment Solution) and import them directly into ATO Copilot, where each plugin finding is parsed, deduplicated, mapped to controls, and optionally escalated to a POA&M weakness entry ŌĆö eliminating manual correlation of thousands of vulnerability findings.

### Key Capabilities

#### .nessus File Parsing (US1)
- XDocument-based parser handles multi-host `.nessus` XML files exported from ACAS
- Extracts all `<ReportHost>` entries with hostname, IP, OS, MAC, scan timestamps
- Each `<ReportItem>` yields plugin ID, name, family, severity, CVEs, description, solution, synopsis
- Malformed XML returns clear error messages with no partial data persisted

#### Automatic Control Mapping (US2)
- **CVE-to-CCI-to-NIST chain**: Plugin CVE references resolve through existing CCI crosswalk to NIST 800-53 controls (confidence: Definitive)
- **Plugin family heuristic fallback**: Curated `plugin-family-mappings.json` maps Nessus plugin families (e.g., "Windows: Microsoft Bulletins") to NIST control families (confidence: Heuristic)
- Only baseline-scoped controls generate `ControlEffectiveness` records; out-of-baseline mappings preserved on findings

#### Severity Mapping & POA&M Auto-Generation (US5)
- Nessus severity levels mapped to RMF CAT severity:
  - Critical (4) / High (3) ŌåÆ **CAT I**
  - Medium (2) ŌåÆ **CAT II**
  - Low (1) ŌåÆ **CAT III**
  - Informational (0) ŌåÆ excluded from findings, counted in summary
- CAT I and CAT II findings automatically generate `PoamItem` weakness entries with:
  - WeaknessSource = "ACAS"
  - Scheduled completion dates (CAT I: 30 days, CAT II: 90 days)
  - Plugin ID, hostname, and CVE references preserved

#### Dry-Run Preview (US3)
- Full import pipeline executes without persistence when `dry_run = true`
- Returns complete preview: finding counts by severity, control mappings, host inventory, POA&M projections
- Enables teams to validate import results before committing data

#### Import History & Re-Import (US4)
- `ScanImportRecord` tracks each import with timestamp, file hash (SHA-256), finding counts, status
- Configurable conflict resolution for re-imports:
  - **Skip**: Existing findings unchanged; new findings added
  - **Overwrite**: Existing findings replaced with latest scan data
  - **Merge**: Existing findings updated only where severity increased
- Duplicate detection via Plugin ID + Hostname + Port composite key
