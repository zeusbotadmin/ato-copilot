## Story

A user whose Entra account is a member of more than one tenant or
organisation sees a tenant picker immediately after authenticating.
They pick a tenant; their session scope locks to that tenant for the
duration. CSP-Admins (Feature 048) see an additional "All Tenants
(CSP view)" option that lands them on the global CSP dashboard. The
picker may be skipped on subsequent sessions via a **device-only
signed cookie**.

## Acceptance scenarios

1. **Given** a user with exactly one tenant, **When** they finish authenticating, **Then** the picker is **skipped** and they land on their default page.
2. **Given** a user with two or more tenants, **When** they finish authenticating, **Then** the picker shows each tenant's display name and status (Active / Suspended / Disabled).
3. **Given** a tenant is `Suspended`, **When** the picker renders, **Then** that tenant shows a yellow "Read-only" badge.
4. **Given** a tenant is `Disabled`, **When** the picker renders, **Then** that tenant is hidden for non-CSP-Admin users and visibly grayed out for CSP-Admins.
5. **Given** a CSP-Admin, **When** the picker renders, **Then** an "All Tenants (CSP view)" option appears at the top and selecting it lands on `/csp/dashboard`.
6. **Given** the user checks "Remember my choice on this device", **When** sign-in succeeds, **Then** a first-party signed cookie scoped to the deployment domain stores the chosen tenant for `Auth:RememberTenantCookieDays`. Clearing browser cookies forgets the choice.
7. **Given** a remembered tenant is later `Disabled`, **When** the user returns, **Then** the cookie is ignored and the picker is shown again.

## Functional requirements covered

- **FR-009**: Picker shown when user is a member of ≥ 2 non-Disabled tenants (or a CSP-Admin).
- **FR-010**: Show each tenant's display name + lifecycle status; hide `Disabled` from non-CSP-Admins; gray out for CSP-Admins.
- **FR-011**: CSP-Admin sees "All Tenants (CSP view)" landing on `/csp/dashboard`.
- **FR-012**: "Remember on this device" issues a first-party HMAC-SHA256-signed cookie; no server-side preference flag.
- **FR-013**: Remembered tenant that later becomes `Disabled` → cookie ignored + picker re-shown.
