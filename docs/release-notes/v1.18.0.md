## Feature 017: SCAP/STIG Viewer Import

ATO Copilot now supports importing **DISA STIG Viewer CKL checklists** and **SCAP Compliance Checker XCCDF results** directly into the compliance pipeline. Imported findings are automatically mapped to NIST 800-53 controls via CCI cross-references and create `ComplianceFinding` records that feed into assessment, remediation, and reporting workflows.

---

### New MCP Tools

| Tool | Description | RBAC |
|------|-------------|------|
| `compliance_import_ckl` | Import DISA STIG Viewer CKL checklist with STIG-to-NIST mapping, conflict resolution, and dry-run preview | ISSM, ISSO |
| `compliance_import_xccdf` | Import SCAP Compliance Checker XCCDF results with automated severity mapping and benchmark detection | ISSM, ISSO |
| `compliance_export_ckl` | Export CKL checklist for DISA STIG Viewer or eMASS upload with finding status synchronization | ISSM, ISSO, SCA |
| `compliance_list_imports` | List import history for a system with filtering by import type, date range, and status | All compliance roles |
| `compliance_get_import_summary` | Detailed per-finding import breakdown with created/updated/skipped/error counts | All compliance roles |

---

### Key Capabilities

#### CKL Import (`compliance_import_ckl`)
- Parse DISA STIG Viewer CKL XML files with full STIG rule extraction
- Map STIG findings to NIST 800-53 controls via CCI cross-references
- Conflict resolution modes: `Skip` (preserve existing) or `Overwrite` (update)
- Auto-determine CAT severity (I, II, III) from STIG rule data
- Create `ComplianceFinding` records with `ScanSourceType=StigViewer`

#### XCCDF Import (`compliance_import_xccdf`)
- Parse SCAP Compliance Checker XCCDF automated scan output
- Extract benchmark metadata (title, version, release date)
- Map scan results to NIST controls through CCI crosswalk
- Create `ComplianceFinding` records with `ScanSourceType=ScapComplianceChecker`

#### CKL Export (`compliance_export_ckl`)
- Generate CKL files compatible with DISA STIG Viewer and eMASS
- Synchronize current finding status into exported checklist
- Support per-benchmark export for targeted assessment review

#### Import Tracking (`compliance_list_imports`, `compliance_get_import_summary`)
- Track all imports with timestamps, user, type, and finding counts
- Per-finding detail with action taken (Created, Updated, Skipped, Error)
- Import history supports audit trail requirements

### Data Model Changes

#### New Entities

- **`ScanImportRecord`** — Tracks each CKL/XCCDF file import with status, finding counts, and metadata
- **`ScanImportFinding`** — Individual finding from a scan import with STIG ID, NIST mapping, and action taken
- **`StigControl`** — DISA STIG rule-to-CCI-to-NIST mapping lookup table
- **`StigBenchmark`** — STIG benchmark metadata (title, version, release date, technology)
- **`CciMapping`** — CCI-to-NIST control cross-reference table

#### New Enumerations

- **`ScanImportType`** — `CKL`, `Xccdf`
- **`ScanImportStatus`** — `Pending`, `Completed`, `Failed`, `PartiallyCompleted`
- **`ImportFindingAction`** — `Created`, `Updated`, `Skipped`, `Error`
- **`ScanSourceType`** — `Manual`, `StigViewer`, `ScapComplianceChecker`

#### Enhanced Entities

- **`ComplianceFinding`** — Added `ScanSourceType` and `ImportRecordId` fields for scan-source tracking
