# v1.29.0 — Unified Security Capabilities Hub (Feature 045)

**Released**: 2026-03-29
**Feature**: 045-capabilities-hub

---

## Overview

Version 1.29.0 delivers the **Unified Security Capabilities Hub** — consolidating CSP profile import, CRM spreadsheet import, and capability management into a single page with a 3-layer model (Components → Capabilities → Control Mappings). This replaces the fragmented import buttons on the Control Inheritance page with a centralized import pipeline and coverage dashboard.

---

## New Features

### 3-Layer Import Model

CSP profiles and CRM spreadsheets now flow through a unified `CapabilityImportService` that creates:

1. **Components** (SystemComponent/Thing) — Provider or technology groupings (e.g., "Azure Government")
2. **Capabilities** (SecurityCapability) — Reusable security solutions (e.g., "Azure AD MFA")
3. **Control Mappings** (CapabilityControlMapping) — Links each capability to NIST 800-53 controls with a role designation

Existing components and capabilities are reused by name match, making imports idempotent.

### CSP Profile Import

Import pre-built CSP profiles from the Capabilities Hub with preview and conflict resolution:

- Select a profile (e.g., "Azure Government — FedRAMP High")
- Preview changes before committing
- Choose Skip or Overwrite for conflicts

### CRM Spreadsheet Import

Upload CSV or Excel CRM files with intelligent column mapping:

- Automatic column detection with sample data display
- Configurable field mapping (Control ID, Inheritance Type, Provider, Customer Responsibility)
- 4-step dialog: upload → column mapping → preview → apply

### Coverage Dashboard

Real-time coverage metrics on the Capabilities page:

- **Provider cards**: Per-provider counts (controlled/total, capabilities, components)
- **KPI bar**: Overall coverage %, total capabilities, gap control count
- **Gap controls table**: Unmapped NIST controls grouped by family

### Component Linking

Multi-select component picker for managing capability-component relationships:

- Search and filter components by type
- Bulk link/unlink with pre-selected indicators
- Component badges on capability cards

### Control Inheritance Simplification

The inheritance page now focuses on per-system designation management:

- Cross-link banner: "Designations derived from Security Capabilities. [Manage Capabilities →]"
- Source capability tooltips showing linked capability names
- CSP Profile and CRM Import buttons removed (moved to Capabilities Hub)

### Guided Onboarding

Empty state component with three action paths for new users:

- Create Manually (blue)
- Import CSP Profile (indigo)
- Import CRM Export (green)

## API Changes

### New Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/capabilities/coverage` | Coverage dashboard metrics |
| GET | `/capabilities/csp-profiles` | List CSP profiles with service counts |
| POST | `/capabilities/import/csp-profile` | Import CSP profile |
| POST | `/capabilities/import/crm` | Import CRM spreadsheet |
| POST | `/components/{id}/capabilities` | Bulk link capabilities |
| DELETE | `/components/{id}/capabilities/{capId}` | Unlink capability |

### Removed Endpoints

| Method | Path | Replacement |
|--------|------|-------------|
| POST | `/inheritance/apply-profile` | `POST /capabilities/import/csp-profile` |
| POST | `/inheritance/import/preview` | `POST /capabilities/import/crm?dryRun=true` |
| POST | `/inheritance/import/apply` | `POST /capabilities/import/crm` |
