## Summary

First-class **Login** experience for ATO Copilot across all four surfaces (Dashboard, VS Code extension, M365 / Teams bot, Web Chat). Replaces the silent-browser-negotiation behaviour that landed users on protected pages without an explicit sign-in flow.

**Closes #68.**

## Engineering notes

- Constitution v2.0.0 check — **PASS** pre- and post-design. No deviations claimed.
- §VI TDD discipline preserved — every implementation task paired with a RED-phase test before GREEN.
- Tenant-scoping invariant preserved — `LoginAuditEvent` is `[TenantScoped]`; pre-session and `NoTenantAssignment` rows use `SYSTEM_TENANT_ID` per clarification Q2 so the automatic query filter applies uniformly.
- FR-033 privacy invariant tested explicitly — `MetadataJson` contains only `{"errorClass":"..."}` shape; no cert thumbprints, serials, subject DNs, or PII beyond `oid`.
- Three-layer simulation gate (config endpoint omit + SPA route guard + endpoint 404) tested at all three layers per research.md § R-Summary item 4.
- Throttle counter (FR-036) is distributed-cache-backed (Redis prod / `IDistributedMemoryCache` dev) so a process restart cannot bypass throttling.
- Schema additions follow Feature 050's `EnsureSchemaAdditions` module pattern instead of `dotnet ef migrations add` (model snapshot drift across F045–F050 would emit cascade mutations).
