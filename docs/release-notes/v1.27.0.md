# v1.27.0 — Visual Compliance Dashboard Platform (Features 030–034)

**Released**: 2026-03-17
**Features**: 030-compliance-dashboard, 031-implementation-roadmap, 032-dashboard-documentation, 033-boundary-scoped-model, 034-dashboard-chat

---

## Overview

Version 1.27.0 delivers the complete **Visual Compliance Dashboard** platform — a full-stack React 19 + .NET 9 solution for real-time compliance monitoring, risk management, and AI-assisted remediation. This consolidated release spans five features:

- **030 — Compliance Dashboard**: Portfolio monitoring, system-level compliance roadmaps, security capabilities catalog, component inventory, gap analysis, and compliance trend analytics
- **031 — Implementation Roadmap**: AI-driven phased remediation planning with Gantt timeline, Kanban bridge, progress tracking, and PDF export
- **032 — Dashboard Documentation**: In-app help panel, contextual tooltips, and expanded MkDocs documentation
- **033 — Boundary-Scoped Model**: Authorization boundary definitions with scoped compliance tracking, composite narratives, and Azure resource discovery
- **034 — Dashboard Chat & Notifications**: SSE streaming chat panel, real-time SignalR notification center, settings panel, and four new dashboard pages

## New Features

### Portfolio Dashboard (030)
- Real-time portfolio table with all registered systems
- Sortable columns: name, impact level, RMF phase, compliance score, ATO countdown, POA&M counts
- ATO countdown severity indicators (green > 90d, yellow 30–90d, red < 30d, expired)
- Filter by impact level and RMF phase
- 15-second auto-refresh polling (pauses on tab blur)

### Single-System Compliance Roadmap (030)
- RMF phase progress stepper (7 phases with completion percentage)
- Control family compliance heatmap (19 NIST families, color-coded)
- Heatmap drill-down showing individual controls per family
- Key metrics: compliance score, ATO status, POA&Ms, narrative coverage
- Finding severity breakdown (CAT I/II/III)
- Activity feed (last 10 events)

### Security Capabilities Library (030)
- Full CRUD for security capabilities (name, provider, category, description, status, owner)
- Control mapping with Primary/Supporting/Shared roles
- Automatic narrative generation per NIST control family
- Narrative propagation on capability updates (skip manually customized)
- Search, category filter, and status filter

### Capability-to-Control Gap Analysis (030)
- Coverage matrix per NIST control family
- Unmapped controls identified per family
- Families below 50% coverage highlighted
- Summary metrics (total/covered/gaps/coverage %)

### System Component Inventory (030)
- People, Places, and Things model for SSP Appendix A
- Component CRUD with capability linking
- Collapsible sections per component type
- Deletion flagging for linked capabilities

### Compliance Trend Analytics (030)
- Background snapshot service (daily at midnight UTC + on-demand after assessments)
- Trend chart with compliance score and narrative coverage lines
- Granularity toggle: Daily, Weekly, Monthly, Quarterly
- Date range presets: 30d, 60d, 90d, 180d, 365d
- Significant decline detection (>5% drop highlighted in red)

### Phased Roadmap Generation (031)
- **AI-driven clustering**: Gaps grouped into logical phases based on severity, control family relationships, and dependency chains
- **Effort estimation**: Per-control effort in person-days using AI analysis with optional historical Kanban task data refinement
- **Risk reduction projections**: Weighted severity scoring (CAT I=10, CAT II=5, CAT III=1) with cumulative risk reduction per phase
- **Dependency sequencing**: Control interdependencies enforced across phases (e.g., AC-2 before AU-6)
- **Deterministic fallback**: Severity-first grouping when AI clustering is unavailable

### Dashboard Roadmap Page (031)
- **Summary metric cards**: Total gaps, estimated effort, projected risk reduction, target timeline
- **Gantt-style timeline**: Horizontal phase bars on a week-based axis with progress fill and status badges
- **Risk reduction curve**: Recharts AreaChart showing projected cumulative risk reduction over time
- **Expandable phase tables**: Per-phase control listings with Control ID, Gap Type, Effort, Role, Dependencies, Status
- **Route**: `/systems/:id/roadmap` with tab link from System Detail page

### M365 Teams Adaptive Cards (031)
- **Roadmap summary card**: Total gaps, phase count, effort, risk reduction, per-phase rows with action buttons
- **Phase detail card**: Items table with Control ID, Effort, Role, Gap Type, Dependencies, Status
- **Action buttons**: Create Kanban Board, Export PDF, Show Phase Details

### Roadmap-to-Kanban Bridge (031)
- **Board creation**: One-click conversion of roadmap phases into a pre-populated Kanban board with tasks
- **Bi-directional sync**: Moving a Kanban task to Done automatically updates the roadmap item and phase progress
- **Task deletion handling**: Deleting a linked Kanban task reverts the roadmap item to NotStarted with warning log

### Roadmap Progress Tracking (031)
- **Actual vs. planned**: Dual-line risk reduction curve (projected + actual) in dashboard
- **Overdue detection**: Phases past their target end week highlighted with red badges and days-overdue count
- **Phase progress bars**: Visual per-phase completion tracking

### Roadmap Restructuring (031)
- **Item moves**: Move controls between phases with automatic aggregate recalculation
- **Phase merge/split**: Combine or divide phases with re-sequencing and recalculated risk points
- **Role and effort updates**: Changes propagate to linked Kanban tasks
- **Empty phase cleanup**: Auto-removes empty phases after item moves

### Roadmap PDF Export (031)
- **QuestPDF rendering**: Header, summary metrics, phase timeline chart, phase detail tables, risk reduction curve
- **Content-Disposition**: Download as attachment from dashboard or MCP tool

### Authorization Boundary Management (033)
- **Boundary definitions**: Create, edit, and delete named security perimeters (Physical, Logical, Hybrid types)
- **Primary boundary protection**: Each system has exactly one primary boundary that cannot be deleted
- **Cascade handling**: Deleting a boundary reassigns resources, components, and mappings to the Primary boundary
- **Dashboard page**: Dedicated boundary management page with summary cards showing resource/component counts and coverage percentages

### Boundary-Scoped Compliance Tracking (033)
- **Scoped capability mappings**: Capability-to-control mappings target specific boundaries for per-boundary compliance tracking
- **Boundary-filtered gap analysis**: Gap analysis page includes boundary selector to filter results by specific boundary
- **Boundary comparison table**: Visual comparison of coverage percentages across boundaries with color-coded indicators

### Composite Narrative Generation (033)
- **Multi-boundary narratives**: Controls with mappings across multiple boundaries auto-generate narratives with organization-wide sections first, then per-boundary sections alphabetically
- **Manual override protection**: Manually customized narratives preserved during propagation (logged as `CompositeNarrativeSkipped`)
- **Per-boundary tracking**: Narrative update response includes counts by boundary

### SSP §11 Authorization Boundary Section (033)
- **Multi-boundary output**: SSP §11 organizes content by boundary definition with name, type, description, resource table, and component inventory per boundary
- **Backward compatibility**: Single-boundary systems render identically to the previous flat format

### Azure Resource Discovery (033)
- **Resource Graph integration**: Discover Azure resources via Resource Graph API
- **Suggested boundaries**: Resources grouped by resource group as suggested boundary definitions
- **Dedup detection**: Resources already in the boundary flagged with badges
- **Dashboard UI**: "Discover Azure Resources" button with search filter, suggested boundary selection, and apply workflow

### Chat Side Panel (034)

#### SSE Streaming Chat
- **Chat panel**: Overlay side panel (420px default, 320–600px resizable) slides in from the right edge
- **SSE streaming**: POST to `/mcp/chat/stream` with progressive token rendering and progress step indicators
- **Markdown rendering**: Full ReactMarkdown with remark-gfm, Prism syntax highlighting (oneDark theme), styled tables
- **Keyboard shortcuts**: Ctrl+Shift+C (Cmd+Shift+C on macOS) to toggle panel, Escape to close
- **Responsive layout**: Full-width overlay below 768px viewport width
- **Focus management**: Auto-focus input on open, focus trap within panel, restore focus on close
- **Drag-to-resize**: Left-edge drag handle, width persisted to localStorage

#### Context-Aware Chat
- **Automatic context injection**: Every chat request includes current page, system ID, boundary ID, RMF phase, and live page metrics
- **Context badge**: Subtle indicator in chat header showing what the AI "sees" (e.g., "Viewing: ACME System")
- **Route-aware updates**: Context refreshes when user navigates between pages mid-conversation

#### Conversation Management
- **Multiple conversations**: Create, switch between, and delete conversations with persistent localStorage history
- **Auto-generated titles**: First message content (max 50 chars) used as conversation title
- **LRU eviction**: Oldest conversations evicted at 50-conversation cap
- **History trimming**: Last 20 messages sent per request for context-aware responses
- **Welcome message**: Phase-aware welcome with RMF phase badge, guidance text, and intelligent suggestion cards

#### Rich Response Rendering
- **Tool evidence panel**: Collapsible section showing executed tools with success/failure icons and execution time
- **Suggestion cards**: Horizontal scrollable chips for one-click follow-up actions
- **Entity link rendering**: Dashboard route links (`/systems/:id`, `/capabilities/:id`) navigate within the SPA
- **NIST control ID detection**: Regex-based rendering of control IDs (e.g., AC-2) as styled linked badges

#### Intelligent Suggestions Engine
- **Phase-aware suggestions**: Context-sensitive "what to do next" based on RMF phase (Prepare through Monitor)
- **Page-specific overlays**: Suggestions based on active dashboard page (boundaries, components, narratives, gap-analysis, etc.)
- **Cross-cutting urgency triggers**: CAT I findings, overdue POA&Ms, and low compliance scores surface regardless of phase
- **Quick action pills**: Always-visible strip above chat input showing top 3 phase+page-aware actions
- **Phase colors and guidance**: Color-coded phase badges with descriptive blurbs and next-phase indicators

## API Endpoints

### Dashboard Portfolio & System (030)

- Portfolio: 1 endpoint
- System Detail: 3 endpoints (detail, heatmap, drill-down)
- Trends: 1 endpoint
- Gap Analysis: 1 endpoint
- Capabilities: 6 endpoints (CRUD + mappings)
- Components: 4 endpoints (CRUD)

### Roadmap Endpoints (031)

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/dashboard/systems/{systemId}/roadmap` | Get active roadmap with phases and items |
| GET | `/api/dashboard/systems/{systemId}/roadmap/progress` | Get progress metrics and risk curve data |
| GET | `/api/dashboard/systems/{systemId}/roadmap/export` | Download roadmap as PDF |

### Notification Endpoints (034)

|--------|------|-------------|
