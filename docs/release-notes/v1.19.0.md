## Feature 018: Security Assessment Plan (SAP) Generation

ATO Copilot now supports **Security Assessment Plan generation, customization, and finalization**. SCAs can generate SAPs from system metadata, customize assessment methods per control, and finalize with SHA-256 integrity hashing. This feature adds 5 new MCP tools and a complete SAP lifecycle management service.

---

### New MCP Tools

| Tool | Description | RBAC |
|------|-------------|------|
| `compliance_generate_sap` | Generate SAP from system metadata with assessment scope, methodology, schedule, and team composition | SCA, ISSM |
| `compliance_update_sap` | Update SAP scope, methodology, schedule, or team composition before finalization | SCA, ISSM |
| `compliance_finalize_sap` | Lock SAP with SHA-256 hash — no further edits allowed after finalization | SCA |
| `compliance_get_sap` | Retrieve SAP details by ID including status, schedule, and team members | All compliance roles |
| `compliance_list_saps` | List all SAPs for a system with status filtering | All compliance roles |

---

### Key Capabilities

#### SAP Generation (`compliance_generate_sap`)
- Auto-populate assessment scope from system baseline controls
- Generate methodology section based on control families and assessment types
- Include team composition from RMF role assignments
- Create assessment schedule with estimated timeline
- Support for Test, Interview, and Examine assessment methods

#### Assessment Method Customization (`compliance_update_sap`)
- Override default assessment methods per control (e.g., change Interview → Examine)
- Customize methodology narrative for organization-specific requirements
- Adjust schedule and scope before finalization
- Track method overrides with justification for audit trail

#### SAP Finalization (`compliance_finalize_sap`)
- Lock SAP content — prevents further edits
- Generate SHA-256 integrity hash of finalized SAP content
- Record finalization timestamp and user
- Required prerequisite before formal assessment begins

#### SAP Lifecycle (`compliance_get_sap`, `compliance_list_saps`)
- View SAP details including status progression (Draft → InReview → Approved → Active → Completed)
- List SAPs with filtering by status
- Track SAP history across multiple assessment cycles

---

### Data Model Changes

#### New Entities

- **`SecurityAssessmentPlan`** — SAP header with title, scope, methodology, schedule, status, and SHA-256 finalization hash
- **`SapControlEntry`** — Per-control assessment method assignment within a SAP
- **`SapTeamMember`** — Assessment team roster entry with name, role, and organization
- **`SapMethodOverride`** — SCA-customized assessment method override with original method, override method, and justification

#### New Enumerations

- **`SapStatus`** — `Draft`, `InReview`, `Approved`, `Active`, `Completed`
- **`AssessmentMethod`** — `Test`, `Interview`, `Examine`
- **`TeamMemberRole`** — `Assessor`, `Lead`, `Observer`, `SystemOwner`, `Isso`, `Issm`
