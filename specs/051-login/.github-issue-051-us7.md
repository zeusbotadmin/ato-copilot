# Feature 051 / US7 — Dev Simulation Login Without Restart (P2)

> **Parent**: #68
> **Phase**: 10 (T120–T129)
> **Story FRs**: FR-023, FR-024, FR-025
> **Quickstart**: [specs/051-login/quickstart.md § 10](../specs/051-login/quickstart.md)

## Status

**Code-complete + automated tests green; manual sign-off (T129) pending.**

## Story

A developer running with `ASPNETCORE_ENVIRONMENT=Development` and
`CacAuth:SimulationMode = true` (027-cac-simulation-mode) sees a
"Simulation Mode — pick an identity" panel on `/login`, listing the
identities defined in `CacAuth:SimulatedIdentities`. This extends
Feature 027 from "single identity per startup" to a curated **list**
the developer can switch between **on the login page** without
restarting the server. The panel is **hard-gated to `Development`
only**: setting `SimulationMode=true` in any other environment is
silently ignored at both the UI and API layers (three-layer gate per
analysis C10: env + flag + non-empty list).

## Acceptance scenarios

1. **Given** `ASPNETCORE_ENVIRONMENT=Development`, `SimulationMode=true`, and 3 configured identities, **When** `/login` renders, **Then** the panel lists all 3 with persona, roles, and tenant.
2. **Given** any non-Development environment (including `Staging`, `Production`, `AzureGov`), **When** `/login` renders, **Then** the simulation panel is **never** shown — even if `SimulationMode=true` is mistakenly left in config.
3. **Given** non-Development, **When** any client `POST`s to `/api/auth/simulate`, **Then** the endpoint returns `404 NotFound` (not 403) and writes a `SimulationBlocked` security audit row.
4. **Given** the developer picks an identity, **When** the SPA calls `POST /api/auth/simulate?identityId=...`, **Then** the server issues a session cookie marked `X-Simulated=true` (separate cookie per analysis C9) and writes the audit row required by Feature 027.
5. **Given** the simulated session is active, **When** any compliance evidence is generated, **Then** the evidence is auto-flagged `IsSimulation=true` per Feature 027 and excluded from real RMF artifact bundles.

## Functional requirements covered

- **FR-023**: Simulation panel renders ONLY when env=Development AND `CacAuth:SimulationMode=true`. Suppress in both SSR config + SPA otherwise.
- **FR-024**: `POST /api/auth/simulate` returns `404 NotFound` (not 403) outside `Development`. Each rejected request writes a `SimulationBlocked` audit row with `severity=Security`.
- **FR-025**: Successful simulation login sets a discrete `X-Simulated=true` cookie (alongside `ato-simulation`) AND tags all evidence with `IsSimulation=true`.
