# Feature 051 / US4 — Login Error States (P1)

> **Parent**: #68

> **Story FRs**: FR-014, FR-015, FR-016
> **Quickstart**: [specs/051-login/quickstart.md § 13](../specs/051-login/quickstart.md)

## Status

**Code-complete + automated tests green; manual sign-off (T084) pending.**

## Story

When authentication fails, the user sees a clear, action-oriented
error page rather than a stack trace, a blank page, or a redirect
loop. Each error class has a dedicated message: smart card not
inserted, certificate expired, certificate not yet valid, certificate
revoked, no tenant assignment, account disabled, MFA failure,
conditional-access block, network failure, and clock skew.
Multi-account Entra selection is delegated to MSAL's built-in account
selector — no custom picker on `/login`.

## Acceptance scenarios

1. **Given** the user is signed into Entra with multiple work accounts, **When** they click the primary sign-in button, **Then** MSAL's built-in account selector is presented. After they pick an account, normal auth proceeds.
2. **Given** no smart card is inserted, **When** the user clicks "Sign in with CAC", **Then** the page shows "Insert your CAC/PIV card and try again" — not a browser-modal certificate prompt loop.
3. **Given** an expired certificate, **When** auth fails, **Then** the page shows "Your certificate expired on {date}. Renew through your sponsor and try again," with no retry button.
4. **Given** the user has no `Tenants` row mapped to their `oid`, **When** auth completes, **Then** the page shows "Your account is authenticated but not provisioned in this deployment. Contact {support email}." Auto-provisioning MUST NOT happen.
5. **Given** a conditional-access block, **When** auth fails, **Then** the page surfaces the Entra-provided remediation URL.
6. **Given** a clock-skew failure, **When** detected, **Then** the page shows "Your device clock is off by {N} minutes. Sync the clock and try again."
7. **Given** a network failure reaching the MSAL endpoint, **When** detected, **Then** the page shows "Could not reach the identity provider. Check your connection and try again." — retry button does NOT spin indefinitely.

## Functional requirements covered

- **FR-014**: Classify CAC failures into `NoCardInserted`, `CertExpired`, `CertNotYetValid`, `CertRevoked`, `ClockSkew` — each rendered with class-specific recovery action.
- **FR-015**: Classify Entra failures into `NoTenantAssignment`, `AccountDisabled`, `MfaFailure`, `ConditionalAccessBlock`, `NetworkFailure`; `ConditionalAccessBlock` surfaces the Entra remediation URL.
- **FR-016**: Multi-account selection delegated to MSAL's built-in selector — `/login` MUST NOT render a custom picker.
